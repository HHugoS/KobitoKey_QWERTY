# GitHub Issue Planner — 作業計画・進行管理支援

## スキル読み込み通知

このスキルが読み込まれたら、必ず以下の通知をユーザーに表示してください：

> 📋 **GitHub Issue Planner スキルを読み込みました**  
> GitHub Issue を活用した作業計画の作成、タスクの管理、および進捗報告をサポートします。

### 🚨 起動時オートロード（AI 向け指示）

このスキルが読み込まれた際、AI は**直ちにリポジトリルートの `.gh-token` ファイルの有無を確認**しなければなりません。
- ファイルが存在する場合、AI はその内容を読み取り、以降の全ての `run_command` 等の操作において、そのトークンを `GH_TOKEN` 環境変数として使用してください。
- これにより、ユーザーが手動で指定しなくても、自動的に適切なアカウントに切り替わります。

---

## 概要

GitHub Issue をプロジェクトの「バックログ」や「作業進行表」として活用し、AIと人間が協調して計画的に開発を進めるためのスキルです。
大規模な変更や複数のステップが必要な作業において、作業の抜け漏れを防ぎ、透明性の高い開発を実現します。

---

## 使用タイミング

- 新しい機能の実装や大規模なリファクタリングを始める前（計画フェーズ）
- 作業を小さなタスクに分解して GitHub Issue に登録したいとき
- 現在の Issue リストを確認して、次に着手すべき作業を特定したいとき
- 実装が完了し、Issue をクローズして進捗を報告したいとき

---

## 手順

### Step 1: 現在のタスク確認

まず、リポジトリ内の未完了（Open）な Issue を確認し、現在の状況を把握します。

1. `gh issue list` で進行中の Issue を一覧表示する。
2. ユーザーが着手したい Issue の番号（ID）を特定する。

### Step 2: 作業計画の自動作成と Issue 登録

ユーザーが「〜を調整したい」「〜を実装して」と要望を伝えた際、AI は**即座にタスクを分解し、Issue の起票を提案**します。

1. **タスク分解**: 要望を実現するためのステップ（AI が行う具体的な操作）を 3〜5 つ程度のチェックリストにまとめます。
2. **起票の提案**: 以下の形式でユーザーに提案し、承認を得たら実行します。
   - **タイトル**: `[Task] 要望の要約`
   - **内容**: 概要、チェックリスト（サブタスク）、関連ファイル。
3. **実行**: `scripts/issue-tool.mjs` を使用して Issue を作成します。

> [!TIP]
> ユーザーが具体的に「Issue を立てて」と言わなくても、新しいプロジェクト性の高い作業が始まる場合は「作業記録のために Issue を作成しましょうか？」と提案することが推奨されます。

### Step 3: Issue 駆動開発 (Branch & PR)

AIは特定の Issue ID をコンテキストとして保持し、以下のフローで開発を進めます。
1. **ブランチ作成**: メインブランチを汚さないよう、`feat/issue-<ID>` のような作業用ブランチを作成します。
2. **実装**: Issue の指示に従ってコーディングを行います。
3. **PR作成**: 作業が一段落したら、プルリクエストを作成してユーザーのレビューを受けます。

### Step 4: 完了報告とクローズ

実装と検証が完了したら、Issue に成果を報告し、クローズします。

```bash
gh issue close <ID> --comment "実装が完了しました。詳細は walkthrough.md を参照してください。"
```

---

### 1. ローカル・アカウントの設定
リポジトリのルートディレクトリに **`.gh-account`** という名前のファイルを作成し、GitHub の **ユーザー名**（例: `HHugoS`）のみを記述します。

> [!IMPORTANT]
> この方式では、トークンをファイルに保存しません。代わりに OS の安全な領域（キーリング）からトークンを動的に取得します。あらかじめ `gh auth login` で対象のアカウントにログインしておいてください。

### 2. 環境変数の反映
以下のコマンドを実行して、トークンをセキュアに読み込みます。

```powershell
. ./.agent/skills/github-issue-planner/scripts/switch-account.ps1
```
※先頭に `.` (ドットとスペース) を付けることで、現在のウィンドウに設定が反映されます。
